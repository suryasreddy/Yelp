README.md
Updated the project overview to separate Lab 1 monolith and Lab 2 Person A stack.
Added docker-compose, Kubernetes, Kafka verification, seeding, and image build/push runbooks.
This works as the main operator guide so evaluators can run infra/services in correct order.
It ties all new files together and explains how to validate health, topics, and async flow.

.env.example
Created a template for MongoDB, JWT, session expiry, and Kafka service variables.
Removed the need for hardcoded runtime values inside service code or manifests.
This works by being copied to `.env` and loaded by Docker Compose services at startup.
It enables identical config keys across local containers and Kubernetes ConfigMap/Secret.

docker-compose.yml
Created a full local stack with Zookeeper, Kafka, MongoDB, 4 API services, and review worker.
Added dependency conditions and healthchecks so startup order is more reliable for demos.
This works by exposing service ports and using compose DNS names for internal communication.
It gives a one-command local environment for API testing and JMeter preparation.

services/requirements.txt
Added shared Python dependencies for all Lab 2 services and worker.
Included FastAPI, pymongo, jose, passlib+bcrypt, and kafka-python libraries.
This works as the single dependency lock for service container builds.
It keeps service Dockerfiles simple and consistent across all backend components.

services/shared/__init__.py
Created package marker for shared reusable Lab 2 modules.
Ensures Python recognizes `services.shared` imports in all service apps.
This works by enabling module resolution with `PYTHONPATH=/app`.
It avoids duplicate code for auth, config, mongo, schemas, and Kafka wrappers.

services/shared/config.py
Created centralized settings loader for Mongo, JWT, session, and Kafka env vars.
Defined defaults suitable for compose/Kubernetes service names.
This works by giving every service the same runtime config contract.
It prevents scattered hardcoded hostnames and makes deployment updates predictable.

services/shared/mongo.py
Implemented Mongo connection reuse and lazy DB initialization.
Added required indexes (email uniqueness, review uniqueness, session TTL, search fields).
This works by calling `get_db()` from services to reuse one client and ensure indexes exist.
It enforces session expiry and common query performance for login/search/review paths.

services/shared/security.py
Implemented bcrypt password hashing and verification helpers.
Implemented JWT creation plus Mongo-backed session document creation and validation.
This works by issuing token with `sid` and checking it against `sessions` collection.
It gives secure password storage and revocable/expiring sessions in MongoDB.

services/shared/http_auth.py
Implemented Bearer-token request authentication for new service endpoints.
Added user lookup and minimal user response sanitizer helper.
This works by decoding token, validating session record, then loading Mongo user.
It standardizes auth checks without reimplementing token logic per service.

services/shared/kafka_bus.py
Implemented shared Kafka producer wrapper with JSON serialization and retries.
Added structured event envelope (type, timestamp, correlation_id, actor, entity, payload).
This works by publishing to topic and waiting for broker acknowledgement.
It ensures review/user/restaurant producer events have consistent metadata.

services/shared/schemas.py
Added Pydantic schemas for auth, preferences, restaurant, and review payload validation.
Kept field naming similar to Lab 1 contracts to reduce frontend incompatibility risk.
This works by validating request payloads before service business logic executes.
It prevents malformed writes and enforces constraints like password/rating ranges.

services/user_reviewer_service/__init__.py
Created package marker for the user/reviewer service module.
Supports clean module import path in uvicorn app startup.
This works with Docker `CMD uvicorn services.user_reviewer_service.app:app`.
It ensures service code remains organized as an importable package.

services/user_reviewer_service/app.py
Created service endpoints for signup, login, profile, preferences, and history.
Migrated auth/session persistence to Mongo with bcrypt and TTL-backed sessions.
This works by writing users/preferences/activity logs and returning JWT session token.
It also publishes `user.created` and `user.updated` events for Kafka-based architecture.

Created dedicated container image build for user/reviewer service on port 8001.
Installs shared requirements and copies service + scripts into image.
This works by launching uvicorn with the service module entrypoint.
It provides isolated deployable runtime for Person A service-splitting requirement.
